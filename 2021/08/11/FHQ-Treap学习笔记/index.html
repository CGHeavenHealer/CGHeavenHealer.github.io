# 简介

FHQ_Treap，是范浩强大佬发明的一种新平衡树，与 Splay 相比，FHQ_Treap 无需复杂的旋转操作，并且常数小；与传统的 Treap 相比，FHQ_Treap 好写好调，支持可持久化，因而广为 OIer 们追捧。

真的好香啊再也不想用 Splay 甚至 STL 了

# 实现

FHQ_Treap 仍然是 Treap，节点内维护的数据和 Treap 是近似的。

这里以小根堆为例。

FHQ_Treap 的基本操作只有两个：分裂、合并。

它的实现基于这样一个显然的事实：从一个点把整棵树拆成两半后，剩下的两棵树依旧是平衡树。而合并操作一般是要在分裂之后进行的，对分裂出的两棵子树进行操作。

分裂：把整个树拆成两部分，左子树的键值小于右子树，之后再进行其他操作

合并与分裂不太一样，需要比较随机优先级，主要目的是保证期望树高为 log⁡n\log nlogn 以保证复杂度。

这样，就可以实现其他平衡树的操作了！

# 用途

  • 插入数据

    插入数据应该是最简单的了，把以该数据为界的左右子树分裂，然后在带着这个数合并即可。

  • 删除数据

    删除时，我们首先把整个树分为两部分：小于等于 xxx 的和大于 xxx 的；之后再把左子树分为两部分：小于等于 x−1x-1x−1 的和大于 x−1x-1x−1 的；那么原来的左子树分出来的右子树满足 val−1<t[x].val≤valval-1 < t[x].val \le valval−1<t[x].val≤val，这就是我们要删除的点了。

    如果有多个点要删除也没有关系，我们可以找到这棵树的根节点，然后让它的左儿子和右儿子合并起来，向原来的树合并，就等于直接无视掉一个值为 valvalval 的元素了。

  • 查排名

    按照 x−1x - 1x−1 的权值把树裂开，左子树中最大的值就应该是 x−1x - 1x−1，所查的数的排名就是 t[a].siz+1t[a].siz+1t[a].siz+1。

  • 查值

    在整棵树和左子树中分别按 xxx 和 x−1x-1x−1 进行分裂，中间部分就是需要的权值。

    或者直接把它写成一个函数，这样 split_sizesplit\_sizesplit_size 就可以丢掉了！

    我们可以仿照线段树二分的方式，在 Treap 上查找排名为 xxx 的点，如果它的 t[ls(x)].siz<xt[ls(x)].siz < xt[ls(x)].siz<x，就往右子树走，否则在左子树上走，直到找到 t[ls(x)].siz+1=xt[ls(x)].siz+1=xt[ls(x)].siz+1=x 的点为止。其实就是找第 kkk 大。

    可以发现其实 split_size 就是一个变相的找第 kkk 大的过程。

  • 查前驱

    需要先将整个树按 x−1x-1x−1 分成两棵子树，再在左子树上查第 t[a].sizt[a].sizt[a].siz 大

  • 查后继

    和查前驱类似

  • 区间反转

    这个应该算得上是平衡树的重头戏了。

    我们需要维护一个反转标记，类似线段树的 Lazy−tagLazy-tagLazy−tag，并将其懒惰下传（注意要在每一个函数开头就执行），在修改时，把整个序列分为三段：[1,l),[l,r],(r,n][1,l),[l,r],(r,n][1,l),[l,r],(r,n]，并将 [l,r][l,r][l,r] 这一段取出来，打上标记后合并，在后面的操作中反转。

# P2042 维护数列

平衡树操作大杂烩。

虽然操作是直接放到同一个题里的，但是绝不仅仅是把这些操作拼一起这么简单，有很多细节要注意：

  • 插入时，要看好 pospospos 的位置以及 tottottot 个数里包不包含 pospospos 位置的数

  • 数列中最多只有 5×1055\times 10^55×105 个数。但是插入操作却有多达 4×1064\times 10^64×106 个，删掉的节点显然没用了，直接搞一个垃圾桶（栈、队列什么的）回收一下，在下次插入时再利用

  • 反转时不仅权值要反转，区间的前缀最大值和后缀最大值也要反转

  • 修改和反转单独处理，打两个标记，不要暴力修改

  • 000 号节点的先设成 −INF-INF−INF，以备后患

  • 注意到这道题选的最大子段和不能选空段，但是左右时可以为空的（也就是只选这一个点）

    所以前缀最大值和后缀最大值要和 000 比一下谁大

  • 修改标记设成一个不可能用到的数，不要直接初始化成 000，因为整个区间覆盖成 000 也是有可能的
