# 前言

# 简介

树链剖分用于将整棵树剖分为若干条链，使它组合成线性结构，然后用其他的数据结构维护信息。

树链剖分可以将树上的任意一条路径划分成不超过 $ log_2n $ 条连续的链，每条链上的点深度互不相同。

定义以下内容：

重子节点 表示其子节点中子树最大的子结点。

轻子节点 表示剩余的所有子结点。

从这个结点到重子节点的边为 重边。

到其他轻子节点的边为 轻边。

若干条首尾衔接的重边构成 重链。

把落单的结点也当作重链，那么整棵树就被剖分成若干条重链。

树链剖分为将树组合成线性结构，需要通过两次DFSDFSDFS 维护以下信息：

  1. faxfa_xfax​ ：每个点的父亲
  2. depxdep_xdepx​ ：每个点的深度
  3. sizxsiz_xsizx​ ：每个非叶子节点的子树大小 (含它自己)
  4. sonxson_xsonx​ ：每个非叶子节点的重儿子编号
  5. topxtop_xtopx​ ：每个点所在重链的链顶

第一次 DFS​：

void dfs1(int u, int F) {
    dep[u] = dep[F] + 1, fa[u] = F, siz[u] = 1;
    for (int i = head[u]; i; i = e[i].nxt) {
        int v = e[i].to; 
		if (v == F) continue;
        dfs1(v, u);
        siz[u] += siz[v];
        if (siz[son[u]] < siz[v]) son[u] = v;
    }
}

处理完以上信息后，我们再 DFS 一遍，处理 topxtop_xtopx​。

顺序：先重后轻（保证 dfs 序是先过重儿子再过轻儿子）

第二次 DFS​：

两次 DFS 的复杂度都是 Θ(n)\Theta(n)Θ(n) 的，这样就把整棵树分成了 log⁡n\log nlogn 条重链。

调用时， dfs1 要注意如果用这种写法， FFF 变量最好设为 000，如果设为负数可能会导致数组越界；而调用 dfs2 时，topftopftopf 要设成 rtrtrt ，因为自己是自己的链顶。

# 应用

  • 求 LCA

    求 LCA 最简单的方法就是一个点一个点的往上跳，倍增法把这个过程优化成一次跳 2k2^k2k 个点；而树剖可以将它优化成一次直接跳到当前重链的链顶，减少了很多不必要的操作

    复杂度 Θ(mlog⁡n)\Theta(m \log n)Θ(mlogn)

    可以先拿这个练练手，熟悉一下：P3884 二叉树问题

  • 与线段树搞比利相配合，把树上两点间链上的问题转化成序列上的问题

    比如，我可以把这样的问题抛给你：

    询问两点间的路径上的节点的最大权值（权值和、最小权值……）。

    修改两点间的路径上的节点的权值（全部加一个数、全部改成另一个数……）。

    询问一个点的子树上的节点的最大权值（权值和、最小权值……）。

    修改一个点的子树上的节点的权值（全部加一个数、全部改成另一个数……）。

    这个应该是树剖最常见的应用了。求个 LCA 啥的都太小儿科了

    按照 DFS 序，我们可以给整棵树重新标号，压成一个序列；可这样又怎么保证区间一定连续呢？

    其实很简单，在 DFS2 中，我们是按照先重后轻的顺序 DFS 的，这样 DFS 序在同一棵子树内是连续的，就可以转成区间问题了。

    那怎样处理路径问题和子树问题呢？

    直接依靠线段树区间修改即可。

    有了这些，你就可以 A 掉这道题了：P3384【模板】轻重链剖分 / 树链剖分

    除了这些，还有一些题，只需要简单的改改线段树维护的东西就可以了：

    P3178 [HAOI2015] 树上操作

    P3833 [SHOI2012] 魔法树

    P2590 [ZJOI2008] 树的统计

    这些都可以用来练手

    看似树剖很长很复杂，其实多打几遍理解起来很快，熟能生巧嘛

    可以默写了以后水树剖题真的很爽很爽很爽！！！

    不过写树剖绝不是提高码力的方式

到这里，树剖结束了吗？

远远没有！

# 技巧 / 槽点

  • 虽然树剖本尊很简单、很模板，但是有时他会稍微改动一下，比如边权转点权、线段树要动态开点等，具体见例题 1、2
  • 有的时候维护的东西不好整体维护，像异或和，这时可以进行拆位，把整个数或者是什么东西变成一个好维护的东西，然后在分别统计贡献。具体见例题 3
  • 线段树处理时一定要注意细节，像 tagtagtag 的初值、标记下传的时机之类

# 1.[SDOI2014] 旅行

就是简单的区间覆盖和区间查询，但是如果对每一个宗教都开一棵线段树会炸的很惨，所以需要用动态开点线段树节约空间。

# 2. 月下 “毛景树”

虽然也是修改、覆盖和查询，但细心的您一定发现这道题的不同之处 ——** 树上的权值从点权变成了边权。** 按照正常的树剖写法，树剖是维护不了边权的，因为两次 DFS 处理的都是点之间的关系 —— 像 DFS 序、链顶、父亲、重儿子之类，而边就不能再这种枚举中找出什么性质。但是，有一种巧妙的方法：我们可以把边上的权值加到深度大的节点上，就像这样：

建立一个类似映射的关系，把边权转成点权，这样就好处理了。

还有一个细节需要注意的，就是根据我们的转换规则，查到链顶时，链顶这个节点存的并不是 x—yx—yx—y 这条链上的数据，而是 xxx 的父亲的数据，所以在查询、修改时不能动它。

# 3. 洛谷树

题目中让我们求两个点路径上所有子路径的异或和，先将点权转成边权，之后我们可以在树上求异或前缀和，显然两个点 x,yx,yx,y 整个路径上的异或和就是 sum[x] xor sum[y]sum[x] \ \text{xor}\ sum[y]sum[x] xor sum[y]。

但是这个所有子链的异或和不好维护，如果一条一条求肯定 T 飞，那怎么搞呢？

考虑到边上的权值特别小，我们可以把每一个数进行二进制拆位，之后通过维护二进制下每一位的值，然后统计起来得到答案，修改就相当于二进制下的区间反转，复杂度 Θ(nlog⁡2n×w)\Theta(n\log^2n\times w)Θ(nlog2n×w)

# 4.[国家集训队] 旅游

这个应该算得上是树剖的究极码农题了，虽然操作并不难

先把点权转成边权，然后分别维护区间最大值和、区间最小值和区间和，这个修改操作非常奇怪，是把一段区间改成相反数，可以打一个反转的标记，最大值、最小值交换进行维护。

说起来很简单，但是这题的坑点很多，像：

  • 关于修改第 iii 条边，要记录边的顺序，修改时比较深度，在深度大的点上改

  • 下传标记时 tagtagtag 要异或

  • 没有标记不要传！！！（这是最重要的）

    一般的线段树操作（区间和、最大值这类）没有标记传了也没事，但是反转标记一定不要随便下传！！！

# 5.[NOI2015] 软件包管理器

也是一道树剖老题了

用 111 表示已安装，000 表示未安装，那么安装软件就相当于从根节点到这个软件的节点的链上赋值为 111，卸载就相当于把这个节点的子树赋值为 000。简单模拟一下即可

要注意的地方：线段树的 tagtagtag 初值不要赋成 0！
