# 虚树

一般情况下，当已经有一棵树且询问次数较多、询问的点较少时，可以对于询问的几个点新建一棵树，这棵树就被称为虚树，原树上被询问的点成为关键点，在树上的点按照 DFS 序排序后，虚树上保存关键点和相邻两个关键点的 LCA

先用树剖和前缀和处理出 LCA 和两点距离的前缀和，建树时用一个栈维护右链。

具体步骤：

  1. 把每个询问的点按 DFS 序排序，将第一个点压入栈中，把栈顶设为 vvv

  2. 枚举下一个点设为 uuu，求出它与栈顶结点的 LCA，设为 lcalcalca

  3. 比较 lcalcalca 与栈顶下一个结点（设为 www）的深度，

    如果 www 的深度大于 lcalcalca，就把 vvv 向 www 连一条边并弹掉 vvv 。重复这步

  4. 如果 LCA 不是 vvv （说明这两个点不在一条链上），就把 lcalcalca 和 vvv 连一条边并弹掉 vvv ，让 lcalcalca 成为栈顶；

    否则让 www 和 vvv 连边

  5. 最后把 uuu 入栈

  6. 从 2 操作开始枚举每一个点

  7. 把栈顶结点向栈顶下一个结点连边，直到只剩下一个点，最后剩下的点就是虚树的根

对 kkk 个点建出的虚树复杂度是 klog⁡nk\log nklogn 的，如果在上面进行 DP 的话复杂度是 Θ(k)\Theta(k)Θ(k) 的。

树链剖分可以将树上的任意一条路径划分成不超过 log⁡n\log nlogn 条连续的链，每条链上的点深度互不相同。

# 笛卡尔树

满足二叉堆的性质：父亲结点比儿子节点更优

将一个序列建成一棵笛卡尔树后，其中序遍历是原序列。

暴力建树：找出最小值，作为根节点，再向左右两个区间扩展，左、右子树的根节点分别是左、右侧的最值。

如果使用 ST 表，复杂度为 Θ(nlog⁡n)\Theta(n \log n)Θ(nlogn) 。

优化：单调栈维护右链（把更劣的元素放在右链上）

# 基环树

有 nnn 个点， nnn 条边的连通图，在树的基础上出现了一个环

环上的每个点都是其子树的树根。

有向图上的基环树分为内向基环树（每个点只有一条出边）和外向基环树（每个点只有一条入边）

# 树上启发式合并

常规数据结构不好维护答案时，可以采用树上启发式合并。

流程：对树做树链剖分，在处理一个节点时，先遍历轻儿子，并解决轻儿子上的问题。之后将其依次删除，重儿子遍历后继承上来，再依次加入轻儿子，做询问。

复杂度 Θ(nlog⁡n)\Theta(n\log n)Θ(nlogn) 。

我们首先观察一下单旋的特点：

首先一步当然是把当前节点 ttt 转到它的父亲 ppp 的位置上，但是直接转的话 ttt 就有了三个儿子，所以我们必须拆出一个儿子去。我们先考虑右旋：

有一个大前提：旋转前后，中序遍历不变。

首先一定不能把 ppp 拆掉，因为由之前树的结构可以看出来， p>tp>tp>t ，所以如果把 ppp 拆掉， ppp 又会回到 ttt 上面，跟没转一样。

那要是拆 aaa 呢？

显然也不行，因为如果把 aaa 拆了，ttt 的两个儿子分别为 b,pb,pb,p ，但是 b>tb>tb>t ， bbb 又只能放在 ttt 的左儿子上，所以不成立。

综上，只能拆掉 bbb ，并把它放在 ppp 的右儿子上。

对于左旋，可以用类似的方法证明。

多 yy 手模几棵树可以看出一个结论：旋转的这个节点的父亲，会被放到原来这个节点相对于它父亲的相反儿子上（例如原来这个点是它父亲的左儿子，它的父亲就会变成它的右儿子），子节点和父亲节点交换位置。

所以，就有了下面的 SplaySplaySplay 操作：

1. 儿子、父亲、祖先在一条链上的：

为了将这棵树由左图的形态旋转到右图，可以连续两次将 xxx 节点向上旋转；

注意，应该先转父亲节点，令祖先节点处于自己父亲节点的位置，再转自己。

否则，如果先转自己，再转父亲节点，父亲节点和自己进行一次旋转，再转自己就又转回来了，跟没转一样。

2. 儿子、父亲、祖先成对勾形的：

只需要进行两次方向不同的单旋即可。

SplaySplaySplay 就是基于这两种情况，然后直到要求的节点转到根节点。

# 插入、删除

插入数据应该是最简单的了，把以该数据为界的左右子树分裂，然后在带着这个数合并即可。

删除时，我们首先把整个树分为两部分：小于等于 xxx 的和大于 xxx 的；之后再把左子树分为两部分：小于等于 x−1x-1x−1 的和大于 x−1x-1x−1 的；那么原来的左子树分出来的右子树满足 val−1<t[x].val≤valval-1 < t[x].val \le valval−1<t[x].val≤val，这就是我们要删除的点了。

如果有多个点要删除也没有关系，我们可以找到这棵树的根节点，然后让它的左儿子和右儿子合并起来，向原来的树合并，就等于直接无视掉一个值为 valvalval 的元素了。

# 查排名

按照 x−1x - 1x−1 的权值把树裂开，左子树中最大的值就应该是 x−1x - 1x−1，所查的数的排名就是 t[a].siz+1t[a].siz+1t[a].siz+1。

# 查值

在整棵树和左子树中分别按 xxx 和 x−1x-1x−1 进行分裂，中间部分就是需要的权值。

或者直接把它写成一个函数，这样 split_sizesplit\_sizesplit_size 就可以丢掉了！

# 点分治

求解树上路径的常用手段

选取树的重心做根，进行分治，分别考虑经过这个点和不经过这个点的路径：对于前者，直接 DFS 求解；对于后者，可以把重心删掉后对子树递归求解。

由于选用的是树的重心作为分治中心，所以每棵子树都不超过树大小的一半，分治层数为 log⁡n\log nlogn ，总复杂度为 nlog⁡nn\log nnlogn 。
