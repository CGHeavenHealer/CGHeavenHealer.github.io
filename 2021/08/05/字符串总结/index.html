# 哈希

对于一个长度为 nnn 的字符串，哈希值为 ∑i=1nSi×basen−imod  p\sum_{i=1}^nS_i\times base^{n-i}\mod p∑i=1n​Si​×basen−imodp，其中 basebasebase 为进制，自选（一般为大于 128128128 的质数）

不计冲突的情况下，可以直接比较两个字符串的哈希值判断其是否相等

为减小冲突概率，可以采用双模数以及自然溢出的方法

# KMP

核心思想：最大限度记录上一次匹配的信息，避免每次都从头匹配。

假如我们匹配的源串匹配到了 iii ，模式串匹配到了 jjj ，每一次匹配到失配的字符时，我们不让 jjj 直接从头开始，而是让它跳到两个串依然相等的前缀，这样就能避免直接返回到开头。

为了避免从头匹配，我们需要定义一个数组 nxtinxt_{i}nxti​：表示在模式串中，长度为 iii 的前缀串中，最大的前缀等于后缀的长度，特别的，如果不存在合法方案的话，令 nxti=0nxt_i=0nxti​=0 。

在求 nxtnxtnxt 时，我们用一个变量 jjj 模拟指针从头开始，如果下一个字符相等，就令 j++j++j++ ，否则就令 j=nxtjj=nxt_jj=nxtj​ 。最后求得的 jjj 就是 nxtinxt_inxti​

匹配时类似，如果 jjj 的下一位能成功匹配，就往下继续匹配；否则就让 jjj 跳到 nxtjnxt_jnxtj​ 继续匹配

# Trie 树

把节点作为状态，把字母放到边上，可以维护多个串、查询某个串是否是这些串的前缀

一般情况下，对于一棵 Trie 树，有以下信息需要维护：

  • tottottot：总的节点数
  • ch[N][M]ch[N][M]ch[N][M]：儿子的信息（第一维和字符串有关，第二维和字符种类有关，ch[i][j]ch[i][j]ch[i][j] 表示 iii 的编号为 jjj 的子节点）
  • cntcntcnt：（视题目而定）附加信息，一般加到叶子节点上（避免和其他串的信息重复）

Trie 树有两种基本操作：插入字符串和查找字符串

# 匹配

查询的时候，先从根节点出发，找到不是完整模式串的一个点后就跳它的 failfailfail 指针，直到不能再跳为止。如果有一个节点已经查到叶子了，就把它的附加信息清空，因为这时说明从根节点到叶子结点，该进行的匹配都已经进行完毕，不需要再跑一遍了

# Manacher

Manacher 可以求解与回文串有关的问题，复杂度是线性的。

对于一个字符串，我们处理出它的 rigrigrig 数组，表示它的最长回文半径。在处理的同时，我们维护一个 pospospos 和 RRR，分别表示当前 rigrigrig 最大时的回文中心以及最大的右边界，回文串的长度为 2×rigi−12\times rig_i - 12×rigi​−1。

有一个性质非常重要：回文串的对称中心两侧的字符相同，两侧关于 pos 成轴对称。

有了这个性质，我们就可以直接借用之前统计出来的答案更新现在的答案。

转移时要注意，当 i≤Ri \le Ri≤R 时，可以直接从 rig[pos×2−1]rig[pos\times 2-1]rig[pos×2−1] 转移答案。注意要判一下边界是不是超过 RRR 了，因为超过 RRR 的部分合不合法是不能保证的。

那么怎么统计超过 RRR 的部分呢？很简单，直接暴力扩展边界即可。

最后如果更新出来的回文串更长，要更新 RRR 和 pospospos。

注意串如果长度不是偶数，要加上一些串中不包含的字符，确保对称中心在字符上。

第 0 个字符一定要设的和其他的都不一样，否则会数组越界死循环！！
