# 简介

最短路径树，顾名思义，就是由最短路的边构成的生成树。

最短路没有不会求的吧

严格定义：对于一个无向连通图，一个以顶点 vvv 构成的最短路树满足以下条件：

  • 在这棵树中，从顶点 vvv 到任意顶点 uuu 的路径距离，是图中 vvv 到 uuu 的最短路。
  • 全图联通，且有 n−1n-1n−1 条边 （毕竟人家是生成树）

SPT 与最小生成树的区别在于，最小生成树不需要满足第一个条件，只要保证全图连通就可以了。

通常情况下，SPT 的边权和 $\ge $ 最小生成树的边权和

SPT 的实现方式主要有 Dijkstra 和 Floyd 两种，但碍于图论题 1e5,1e61e5,1e61e5,1e6 等吓人的数据，以及 Dijkstra 本身算法原理的优点（按照贪心原理，把一条条边从图中拽出来），实现它通常用 Dijkstra 。

我们可以用一个 preprepre 数组记录下松弛操作后边的前驱，即可还原出一棵生成树。

复杂度： Θ(n+m)log⁡n\Theta (n+m)\log nΘ(n+m)logn

（除了记前驱的一步外和普通的 dijkstra 一模一样）

# 应用

# 1.CF1076D Edge Deletion

给一个 nnn 个点， mmm 条边的无向简单带权连通图，，要求删边至最多剩余 kkk 条边.

定义 "好点" 是指删边后，，1 号节点到它的最短路长度仍然等于原图最短路长度的节点.

最大化删边后的好点个数。

# 分析：

由定义可以得出，好点其实就是 SPT 上的点。

由此可得，我们可以把 1 号节点当作根节点，建最短路树，从 1 号节点开始 DFS，遍历 min{n,k+1}min\{n,k+1\}min{n,k+1} 个节点，保证遍历后的结果构成一棵树即可。

# 2.CF545E Paths and Trees

给定一张带正权的无向图和一个源点，求边权和最小的最短路径树。

# 分析：

只需要套板子求出前驱数组，然后再依次累加边权即可

由于我们使用的是堆优化的 Dijkstra ，所以率先被扩展出来的点的 disdisdis 一定是更小的。

在最短路相等的情况下，扩展到同一个节点时，后出堆的点连的边权值一定更小

3.CF1005F Berland and the Shortest Paths

求最短路树的方案

# 分析：

对于一个方案，可以在记录前驱的时候把它放到一个决策集合里，方案数就是每个集合的元素乘积

注意要判断一下方案数与 kkk 的关系，而且不能都执行完乘法在去判断，原因是毒瘤的 hack 者搞了一组溢出到 000 的数据 （丧心病狂）

单位权值，所以可以直接广搜。

输出答案时，dfs 逐情况枚举直到超过输出限制即可。
