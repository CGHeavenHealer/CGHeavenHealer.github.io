# T1

「雅礼集训 2018 Day1」养花 20210903-1

分块。

对于每一个数 xxx，贪心的想：xmod  kx\mod kxmodk 的最大值最大时一定是 x−1x-1x−1，取不到的话会是 x−2,x−3…x-2,x-3\ldotsx−2,x−3…，这个东西我们可以先处理出前缀最大值，之后用一个桶维护。

有了这个，我们可以预处理出对于每个模数，每个块内的最大值，之后的询问就是分块基础操作了。

# T2

DP

设 f[i][0/1]f[i][0/1]f[i][0/1] 表示在 iii 这个点，向左上 / 右上走的方案数，由于对 yyy 排序后， xxx 的关系不好表示，所以我们直接对 xxx 排序，之后判断 yjy_jyj​ 是否在 yiy_iyi​ 上面即可。

转移方程：

  • 当 yj>yiy_j>y_iyj​>yi​ 时， f[j][1]+=f[i][0]f[j][1]+=f[i][0]f[j][1]+=f[i][0]；
  • 否则 f[i][0]+=f[j][1]f[i][0]+=f[j][1]f[i][0]+=f[j][1]。

初始状态为 f[i][0]=f[i][1]=1f[i][0]=f[i][1]=1f[i][0]=f[i][1]=1。

最后答案为 (∑i=1nf[i][0]+f[i][1])−n(\sum_{i=1}^nf[i][0]+f[i][1])-n(∑i=1n​f[i][0]+f[i][1])−n，因为每一个点单独的方案都被计算了两次。

#include <bits/stdc++.h>
using namespace std;
#define int long long
#define ri register int
const int N = 6010, mod = 1e9 + 7;
inline int read() {
	ri x = 0, f = 1;
	char c = getchar();
	for (; !isdigit(c); c = getchar())
		if (c == '-') f = -1;
	for (; isdigit(c); c = getchar()) x = (x << 1) + (x << 3) + (c ^ 48);
	return f * x;
}
int f[N][2];
struct node {
	int x, y;
	inline bool operator<(const node &a) const { 
		return x == a.x ? y < a.y : x < a.x;
	}
} a[N];
signed main() {
	int n = read(), ans = 0;
	for (ri i = 1; i <= n; i++) a[i].x = read(), a[i].y = read();
	sort(a + 1, a + n + 1);
	for (ri i = 1; i <= n; i++) {
		f[i][0] = f[i][1] = 1;
		for (ri j = i - 1; j; j--) {
			if (a[j].y > a[i].y) (f[j][1] += f[i][0]) %= mod;
			else (f[i][0] += f[j][1]) %= mod;
		}
 	}
 	for (ri i = 1; i <= n; i++) (ans += (f[i][0] + f[i][1])) %= mod;
 	printf("%lld\n", (ans - n + mod) % mod);
	return 0; 
}
/*
4
2 2
3 1
1 4
4 3
*/

# T3

字符串

题意非常诡异：求出给定字符串的长度为 xxx 的一个前缀是否满足它的一个前缀翻转过来的前缀能和这个串的后缀匹配，能的话输出 xxx。

考虑一个既暴力又有问题的算法：处理出原串正和反的哈希之后，n2n^2n2 扫描前缀以及前缀的前缀，之后 Θ(1)\Theta(1)Θ(1) 匹配反转后的前缀和原串的后缀。

这样就有问题了：因为这个串的前缀可以无限次反转拼接，所以像这样的一个串 qwqqwqqwq，可以得到 qwqwq,qwqwqwq…qwqwq,qwqwqwq\ldotsqwqwq,qwqwqwq…，直接判断它可能并不能得到合法方案。

所以我们把正着扫改成倒着扫，如果扫的位置在字符串的后一半，它是不可能有循环节的，直接判断即可。注意这里的长度上限是 n−i+1n-i+1n−i+1，因为前缀的二倍一定比整个串长了；而如果扫的位置在前一半，考虑他反转过去，对应点是否合法，以及它到反转过去的点这一段能否成功匹配。
