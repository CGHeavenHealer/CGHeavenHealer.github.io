# 简介

主席树，全称是可持久化权值线段树。

权值线段树，和普通线段树不同的是，它存储的 [l,r][l,r][l,r] 这个区间不表示 l−rl-rl−r 这个区间的信息，而是整个区间在 l−rl-rl−r 这个值域内的信息，诸如有多少个数之类。每次在扫到一个数时，我们应该二分这个数的区间，之后在指定位置插入这个数。

但是如果想维护历史版本呢？显然不能建 mmm 棵线段树。可以发现每次线段树的单点修改只会修改 log⁡n\log nlogn 级别的点，所以我们可以在这些点另开副本，其它点沿用历史版本即可。如图：

可以发现，如果一个一个把数插入到主席树里，主席树的整个结构是不会改变的，只有对应节点的值会受影响。所以，值域相同时 ** 主席树（包括权值线段树）是可差分的。** 这点非常重要。

# 用途

由于是权值线段树，所以它可以完成权值树的一切工作，最常见的是查询区间第 kkk 小：【模板】可持久化线段树 2

每次加入一个数，就新开一个主席树版本，这样的空间是 nlog⁡nn\log nnlogn 的。由于我们在树上维护了 l−rl-rl−r 值域内的数字个数，所以只需要比较 cntl−r,kcnt_{l-r},kcntl−r​,k 的大小关系，即可在线段树上二分求解，时间也是 nlog⁡nn\log nnlogn 的。

而对于区间 l−rl-rl−r，其实就相当于对第 l,rl,rl,r 棵树操作。求区间第 kkk 小，就相当于在第 rrr 个版本和第 l−1l-1l−1 个版本的差值树上找第 kkk 小。因为值域树是可差分的，这棵差值树对应的就是 l−rl-rl−r 这个区间有多少个数落在值域区间内。

# 例题

# [SDOI2009] HH 的项链

求区间 [l,r][l,r][l,r] 内不同颜色的个数。

我们可以记录一个 preprepre 数组表示这个值在这个位置之前最后出现的位置，问题可以转化为在 [l,r][l,r][l,r] 这个区间，有多少个数满足 pre[i]<lpre[i]<lpre[i]<l。这个信息可以直接用主席树维护：每次在主席树上插入 pre[i]pre[i]pre[i]，之后每次询问找 pre[i]<lpre[i]<lpre[i]<l 的数的个数即可。

# Count on a tree

查询 u,vu,vu,v 两点间第 kkk 小的权值。

这个只是把区间第 kkk 小放到了树上，处理方式有一些改变：差分时，从原来的 rt[r]−rt[l−1]rt[r]-rt[l-1]rt[r]−rt[l−1] 变成了 rt[u]+rt[v]−rt[lca(u,v)]−rt[fa[lca(u,v)]]rt[u]+rt[v]-rt[lca(u,v)]-rt[fa[lca(u,v)]]rt[u]+rt[v]−rt[lca(u,v)]−rt[fa[lca(u,v)]]，加上树剖预处理即可。

# [POI2014]KUR-Couriers

查询一个区间内有没有数的出现次数严格大于一半。

这个直接在树上找第 ⌈(r−l+1)2⌉\lceil \dfrac{(r-l+1)}{2} \rceil⌈2(r−l+1)​⌉ 大的值就好了。注意要是找到一个节点，他的权值已经小于一半了，直接返回 000 即可。

# 数颜色

写一种数据结构，支持区间查询一个数的出现次数，交换两个数。

区间查出现次数直接做就好，交换两个数可以用差值的思想暴力换，先给这个数减掉它现在的值，再加上要换的那个数的值。并且不需要修改后续版本。

# [SHOI2007] 园丁的烦恼

查询以 (a,b)(a,b)(a,b) 为左下角，(c,d)(c,d)(c,d) 为右上角的矩形内部（包括边界）有多少个点。

先把所有点按横坐标升序，之后把纵坐标插入，可以把问题变成求 a≤x≤c,b≤y≤da\le x\le c,b\le y\le da≤x≤c,b≤y≤d 的点数。

对每个询问，我们需要二分出 a,ca,ca,c 对应的点的范围 l−rl-rl−r，之后查询 b,db,db,d 的答案，相减即可

Dynamic Rankings

这个东西只是把静态第 kkk 小改成了动态。但恶心程度却把静态的甩了八条街

如果不带修改的话，静态的主席树解区间第 kkk 小要用到前缀和。主席树已经没有什么优化空间了，考虑在这个前缀和上动动手脚。

我们可以用一个树状数组或线段树维护每一棵主席树，也就是树状数组的每个节点都是一棵主席树。记录它的前缀和，同时支持单点修改。

对于每个修改，可以直接在树状数组上修改对应主席树的节点；

而对于查询操作，需要先在树状数组上把对应区间 l−rl-rl−r 的节点搞出来，然后查询时需要在这些节点上同步遍历。

数据规模太大，需要离散化。
