# 定义

仙人掌图，指的是一张任何一条边都最多在一个简单环上的无向连通图。

# 应用

这里以 P5236【模板】静态仙人掌 为例。

题意：在一个仙人掌图上，有 qqq 次询问，回答给定的两个点之间的最短路。

由于仙人掌的性质很好（每条边只能最多在一个环上），所以处理起来会有一定的简化手段，常见的是建圆方树。

什么是圆方树？

圆方树，其实就是把原来仙人掌上那些可能有的环拆掉，在原图之外对每一个环建一个方点，原图上的点都是圆点，方点向对应的环的圆点连边，建成的树。

简单想一想，这是棵树应该还是挺显然的。

比如上面那张图，建出的圆方树就长这样：

建树的过程：先按 tarjan 处理出 dfndfndfn 和 lowlowlow 数组，按 DFS 序建出新的图。这里由于是仙人掌图，所以有一个特殊的性质，就是返祖边覆盖的区间是没有重叠的，处理起来更简单一点：

之后处理环：每个环需要处理环上的最短路，并且建好圆方树。

处理好环以后，可以把问题转成树上的最短路。

先对新图树剖一下，求出 LCA，用深度最小的点代表整个环，记为 xxx，那圆方边的边权就是圆点到 xxx 的的最短距离。

对 lca(x,y)lca(x,y)lca(x,y) 的情况分类讨论：

  • 如果其为圆点，那两点间的最短距离就是 disx+disy−2×dislca(x,y)dis_x+dis_y-2\times dis_{lca(x,y)}disx​+disy​−2×dislca(x,y)​；
  • 如果其为方点，先找出 xxx 的两个儿子 son1,son2son1,son2son1,son2，分别为 x,yx,yx,y 的祖先。因为 son1,son2son1,son2son1,son2 都在一个环上，距离可以直接预处理出来；最短的距离就是 dis(son1,x)+dis(son2,y)+dis(son1,son2)dis(son1,x)+dis(son2,y)+dis(son1,son2)dis(son1,x)+dis(son2,y)+dis(son1,son2)。

这里树剖处理第二种情况比较复杂，需要沿着轻儿子一直向上跳，直到两个点都在一条链上；如果正好跳到 xxx，链上它的轻儿子就是要找的节点。

剩下的就是把这些加起来，就可以了。
